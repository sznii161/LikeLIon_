- 지도 기반 카공앱의 기본 방향을 다시 확인했다.
- 현재까지 정리된 데이터를 바탕으로 성북구 카페 정보를 카공앱 기준에 맞게 업데이트하는 작업이 필요하다고 정리했다.
- 지도는 기본 화면으로 두고 사용자가 원하는 기준에 따라 근처 카페를 우선순위별로 확인하거나 등급별로 볼 수 있는 구조를 논의했다.

## 2. 오늘의 안건

### Agenda 1. 화면 기본 구조
- 앱은 지도를 기본으로 보여주는 방식으로 진행하기로 했다.
- 지도 위에서 사용자의 기준에 맞는 근처 카페를 우선순위별로 보여주거나 등급을 설정해 보여주는 방식을 검토했다.
- 이 구조는 사용자가 원하는 조건에 맞는 카페를 빠르게 찾는 데 초점을 둔다.

### Agenda 2. 데이터 업데이트 방향
- 정리해 온 데이터를 바탕으로 성북구 카페 정보를 카공앱 기준에 맞게 다시 정리하기로 했다.
- 단순 목록 제공이 아닌 앱에서 사용하는 점수 기준과 필터 기준에 맞게 데이터 정리가 필요하다고 논의했다.
- 이후 사용자가 선택한 카테고리에 따라 원하는 카페를 보여주는 기능을 핵심 작업으로 잡았다. 

### Agenda 3. 개발 우선순위
- 현재는 기능을 더 확장하기보다 먼저 기본적인 틀을 만드는 것이 우선이라는 의견이 나왔다.
- 구현 담당은 기본 화면과 구조를 다음 회의 전까지 먼저 만들어 보겠다고 했다.
- 구현 후에는 다음 회의에서 진행 상황을 공유하고 세부 기능을 조정하기로 했다. 

### Agenda 4. 일정 조율
- 다음 주 평일은 대부분 가능하다고 공유했다.
- 월요일은 어렵고 수요일 오후 4시가 적절하다는 데에 의견이 모였다.
- 회의 종료 후 다음 일정은 수요일 4시로 확정하는 흐름으로 정리됐다.

- 정리한 데이터를 바탕으로 성북구 카페 정보를 앱 기준에 맞게 업데이트한다. 
- 사용자가 선택한 카테고리에 따라 카페를 보여주는 기능을 핵심으로 한다.
