## 1. 오늘의 논의 내용

이번 회의에서는 프로젝트의 배포 방식, 발표 방식, 영상 제출 방식, 발표 분담, 그리고 시스템 아키텍처 준비 사항을 정리했다.  
이미 주요 기능은 대부분 구현이 완료된 상태이며, 최종 발표 준비와 제출 방식 조율이 중심 안건이었다.

### Agenda 1. 배포 방식 및 제출 방법
- 배포는 박영욱님이 진행하기로 했다.
- 배포 관련 세부 내용은 카카오톡 또는 최종 회의 때 공유하기로 했다.
- 배포가 진행되면 GitHub에 올린 내용을 기준으로 확인할 예정이다.

### Agenda 2. 발표 및 영상 제출 방식
- 발표는 프론트와 백엔드를 나누어 진행하는 방향으로 정리했다.
- 영상은 각자 따로 촬영한 뒤 합쳐서 제출하는 방식이 깔끔하다는 의견이 나왔다.
- 영상 제출은 하나로 모아 올리는 방식을 우선 고려하되 구글 드라이브를 활용한 공동 업로드도 가능하다고 논의했다. 

### Agenda 3. 발표 내용 분담
- 각자 자신이 맡은 부분 중 중요하다고 생각한 내용을 중심으로 발표하기로 했다.
- 박영욱님은 PM 역할로서 프로젝트 전체 소개와 마무리 설명을 맡는 방향으로 정리했다.
- 발표에서는 시스템 구조, 채점 방식, 등급 매기는 기준, API 명세와 데이터 송수신 방식 등을 포함하기로 했다.

### Agenda 4. 시스템 아키텍처 자료 준비
- 제출용으로 시스템 아키텍처를 그래프 또는 마인드맵 형태로 만들어야 한다는 이야기가 나왔다.
- 해당 자료는 차성진님이 만들어보기로 했다.
