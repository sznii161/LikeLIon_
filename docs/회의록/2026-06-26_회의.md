## 1. 오늘의 논의 내용

### Agenda 1. 프론트 진행 상황
- **논의 내용:**
  - 내 위치 확인 버튼을 구현함.
  - 로그인 UI를 구현함.
  - 카페 상세 정보 표시와 리뷰 작성 기능을 구현해둔 상태.
- **결정:** 현재 구현된 기능을 기준으로 이후 연동 작업을 진행하기로 함.

### Agenda 2. 리뷰 데이터 연동 방식
- **논의 내용:**
  - 카페 리뷰 데이터를 방문 리뷰에 올릴지에 대해 확인함.
  - 기존에 수집한 리뷰 데이터는 카페 정보 업데이트용으로 모은 자료라는 점을 확인함.
  - 현재 지도 서비스 안에서는 사용자가 직접 리뷰를 남기는 방식으로 가기로 함.
- **결정:** 기본 구조는 유저 리뷰 작성 방식으로 유지하고, 기존 리뷰 데이터도 함께 연동하는 방향을 검토하기로 함.
- **근거:** 화면이 비어 보이지 않도록 기존 리뷰를 함께 보여주는 것이 더 자연스럽고 완성도 있어 보이기 때문임.

### Agenda 3. 위치 기본값 처리
- **논의 내용:**
  - 내 위치 기반 시작 시 성동구가 아니라 천안으로 잡히는 문제가 확인됨.
  - 발표 영상 촬영 시에는 하드코딩으로 위치를 고정하는 방법을 사용하면 좋겠다는 의견이 나옴.
- **결정:** 발표용 화면에서는 위치를 고정하는 방식으로 처리하기로 함.
